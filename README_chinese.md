# CSS 弹性盒子 — 第二部分：根据设计稿编写样式

你已经通过取消注释的方式探索了弹性盒子的功能。

现在轮到你**自己来写**了。

你将获得：
- 第一部分相同的 `index.html` 文件 — 不要修改它
- 一个新的 `style.css` 文件，其中有一些属性**缺失**
- 一张设计稿，显示你完成后页面应该是什么样子

> **你的目标：** 填写缺失的 CSS 值，让你的页面与设计稿完全一致。

---

## 开始之前

并排打开这三样东西：

1. Codespaces 中的 `style.css`
2. 实时预览（右键点击 `index.html` → Open with Live Server）
3. 设计稿参考页

在写任何内容之前，仔细观察设计稿。

**问问自己：**
- 页眉是什么颜色？
- 导航链接在哪里？
- 两个区块朝哪个方向排列？
- 页脚文字在哪里？

---

## 与第一部分有什么不同？

颜色和间距发生了变化。一些弹性盒子属性的**值**也改变了。

以下是新的变化：

| 区域 | 改变的内容 |
|------|-----------|
| 页眉 | 深青色背景。文字现在在**左侧** — 不再居中 |
| 导航栏 | 绿色背景。链接从**左侧**开始 — 不再居中 |
| 主内容区 | 更暖的米白色背景。间距更小（16px 而不是 24px）|
| 页脚 | 文字推到了**右侧** — 不再居中 |

## 缺失的属性

你的 `style.css` 有 **5 个空白**需要填写。它们用 `/* ??? */` 标记。

找到每一个，并用正确的值替换它。

### 空白 1 — `.header` → `align-items`

看看设计稿。标题和副标题在页眉的**左侧**。

哪个值可以让内容靠左对齐？

```css
.header {
  align-items: /* ??? */;
}
```

**提示：** 不是 `center`。查看弹性盒子速查表，找其他选项。

---

### 空白 2 — `.nav` → `flex-direction`

导航链接**并排**排成一行。

哪个 `flex-direction` 值能让内容从左到右排列？

```css
.nav {
  flex-direction: /* ??? */;
}
```

**提示：** 你在第一部分用过这个值。检查你的笔记。

---

### 空白 3 — `.nav` → `justify-content`

链接从导航栏的**左边缘**开始排列。

哪个值能把内容推到起点？

```css
.nav {
  justify-content: /* ??? */;
}
```

**提示：** 它是 `flex-end` 的反义词。内容会去到开头的位置。

---

### 空白 4 — `.main` → `display`

两个区块**并排**放置。

要让子元素并排排列，需要给父元素添加什么？

```css
.main {
  display: /* ??? */;
}
```

**提示：** 这是唯一一个能开启弹性盒子的属性。今天你已经写过很多次了。

---

### 空白 5 — `.footer` → `justify-content`

页脚文字在**右侧**。

哪个值能把内容推到末尾？

```css
.footer {
  justify-content: /* ??? */;
}
```

**提示：** 它是 `flex-start` 的反义词。想想一行内容的末尾在哪里。

## 检查你的作业

填完所有 5 个空白后，你的页面应该与设计稿完全一致。

逐项检查：

- [ ] 页眉背景是深青色 `#1a3a4a`
- [ ] 页眉文字在左侧

- [ ] 导航链接从左侧开始，之间有间距

## 遇到困难了？

在请求帮助之前，先试试这些：
